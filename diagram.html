How the System Works

  • The simulation starts by creating a queue of requests based on the number of servers.
  • Each request includes an input IP, an output IP, and the time needed to finish.
  • The load balancer checks for empty servers every clock cycle to assign new jobs.
  • If the queue gets too long, the system adds a new server to help with the load.
  • If the queue gets too short, the system removes a server to save resources.
  • The servers will process the jobs, and increment the clock cycle.
  • All events and changes are recorded in a log file for review.
