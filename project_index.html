Load Balancer Simulation Project

This project simulates a large company's load-balancing system for managing web requests. The load balancer will dynamically allocate and deallocate web servers, trying to keep the request queue balanced between 50-80 requests per server.

Features: Request generation with random IP addresses and processing times, dynamic server management, IP range blocking for firewall/DOS prevention, and logging with color-coded output.
